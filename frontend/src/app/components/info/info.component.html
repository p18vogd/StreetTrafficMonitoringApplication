Traffic Data API Documentation

Overview

The Traffic Data API provides information about traffic on various streets. The data includes the number of cars, timestamps, average speed, sensor information, street information, and street names.

Base URL

The base URL for the API is http://localhost:8082.

Endpoints

Get data for a specific time period

The base URL for the API is http://localhost:8082/gov.

Data for streets with highest congestion

Retrieves and returns a list of the ten streets with the highest car count within the specified date range. The list is sorted

GET : http://localhost:8082/getTenHighestCarCountStreets.

Data for streets with medium/low congestion

Retrieves and returns a list twenty streets with the medium or low car count within the specified date range. The list is sorted

GET : http://localhost:8082/getLowCongestionData.

Average data that covers road network

Retrieves and returns a list of the average speed in road network, the number of all vehicles on street the number of sensors that was available for that day and the number of streets that was monitored

GET : http://localhost:8082/getMiniCardData.

Parameters

All endpoints require parameters to determine the time period you want the data to be retrieved. Those params are the following

  • startDate (String): Start date for the data retrieval ( Format: DD/MM/YYYY )
  • endDate (String): End date for the data retrieval ( Format: DD/MM/YYYY )

Example

The following example is demonstrating the way to retrieve traffic data using the /gov endpoint. This endpoint will return a large json response containing all the info about the traffic condition from March 3 2022 to March 5 2022

GET : http://localhost:8082/gov?startDate=03/03/2022&endDate=05/03/2022
